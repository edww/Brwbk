# Ideas

Carnet d’idées Brewbook. Rien n’est supprimé pour pouvoir revenir sur une piste plus tard.

## Identité

- Garder le nom Brewbook très visible.
- Utiliser le même logo partout.
- Ne pas réduire l’identité à un simple `b` si cela rend le nom moins mémorisable.
- Interface blanche, rose comme accent, le café visible surtout dans les photos.

## Notation

- Note globale par extraction.
- Notes par méthode sur chaque café : Espresso, V60, Aeropress, French Press, Cold Brew, etc.
- Afficher uniquement les notes existantes.
- Moyennes automatiques par méthode.
- Note personnelle : `est-ce que je rachète ce café ?`
- Plus tard : commentaires guidés après une note basse.

## Aides pédagogiques

- Boutons `i` pour chaque critère.
- Définitions courtes.
- Exemples simples.
- Confusions fréquentes : acidité vs amertume, amertume vs brûlé, corps vs concentration.
- Mini coach de dégustation.

## Fiche café intelligente

- Meilleure méthode.
- Méthodes testées.
- Nombre d’extractions.
- Dernière extraction.
- Meilleur shot.
- Profil dominant.
- Est-ce qu’il fonctionne mieux en espresso ou en filtre ?

## UX

- Recherche instantanée.
- Filtres par origine, process, torréfacteur, méthode.
- Dupliquer une extraction.
- Partir d’une recette existante.
- Comparer deux shots.
- Ne jamais afficher de données vides.

## Technique

- Garder JPEG pour les photos.
- Plus tard : 800 px max, qualité 75–80 %.
- Étudier IndexedDB si beaucoup de photos.
- Garder export JSON très visible.
- Ne pas casser les données avec les changements de version.

## Futur lointain

- Scanner un paquet.
- OCR des infos du café.
- Base de torréfacteurs.
- Partage de recette.
- Export PDF.
- Synchronisation cloud.
- Version App Store.
